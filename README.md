General useful features, that are not commands:
- For tickets you can click both cancel, and complete ticket, even if you are not the requester.
!!!! IF THE TICKET HAS LESS THAN MAX HELPERS AND PEOPLE HAVE BEEN PERMA YAPPING IT WILL BE HARD FOR THE CONFIRM BUTTON TO BE SEEN (when you click jump to present messages it won't be there). There are 2 possible fixes for it:
1. Remove/delete enough messages for it to work.
2. Click the button and quickly quickly scroll down/click on jump to present messages so that when the response is sent you are ready to click it.

Commands available for economy:
- `/purse-adjust <user> <coins>`
  - Adjust a users coins. Positive values (5) give coins while negative values (-5) subtract coins. 

Commands available for tickets:
- `/adjust-points <user> <points>`
  - Same as for purse-adjust, just for ticket points.
- `/clear-active-ticket <user> <ticket-{id}>`
  - Sometimes if people claim while a ticket is being cancelled it may bug out and they get added as claimer even though they should not. Use this command to fix those cases. I think you should be able to not use ticket-id, but if you need it its just ticket-{id}, example: ticket-34.
- `/remove-claimer <user> <ticket-{id}>`
  - For cases where someone does not participate and they should not be rewarded points, use this to remove them from the ticket in question. The formatting of how you use it is the same as for clear-active-ticket.
